# ADR 0003: MVP Scope

## Context

The product has many possible future modules, but the first release must validate real usage without becoming a full ERP.

## Decision

The MVP includes only:

- Core condominium management.
- Official communication.
- Tickets and occurrences.
- Documents.
- Common area reservations.
- Simple syndic dashboard.

## Alternatives Considered

- Include financial billing and bank integrations.
- Include formal digital assemblies.
- Include native apps, full gatehouse, and AI features.

## Consequences

The product starts safer and simpler. Future modules can be added only after the core tenant and permission model is proven.
